# ToyCompany API

MVP de API REST em `.NET 8` para gerenciamento de brinquedos infantis com arquitetura limpa, principios SOLID, Entity Framework Core e configuracao para Oracle Database.

## Objetivo

O projeto atende uma empresa de brinquedos para criancas de ate 14 anos, expondo endpoints HTTP para CRUD completo da entidade `Brinquedo`.

Campos previstos na tabela:

- `Id_brinquedo`
- `Nome_brinquedo`
- `Tipo_brinquedo`
- `Classificacao`
- `Tamanho`
- `Preco`

## Arquitetura

O projeto foi organizado em camadas para reduzir acoplamento e facilitar manutencao:

- `ToyCompany.Domain`: entidade `Brinquedo` e regras de negocio.
- `ToyCompany.Application`: contratos, DTOs e servico de aplicacao.
- `ToyCompany.Infrastructure`: `DbContext`, mapeamento EF Core para Oracle e repositorio.
- `ToyCompany.Api`: endpoints, Swagger e middleware de tratamento de erros.
- `ToyCompany.Tests`: testes unitarios com xUnit.

## Estrutura da solucao

```text
ToyCompany.sln
|-- ToyCompany.Api

```

## Banco Oracle

No GitHub, mantenha apenas um valor ficticio em `ToyCompany.Api/appsettings.json` ou `ToyCompany.Api/appsettings.Development.json`:

```json
"ConnectionStrings": {
  "OracleConnection": "User Id=SEU_USUARIO;Password=SUA_SENHA;Data Source=localhost:1521/XEPDB1;"
}
```

Para desenvolvimento local, prefira configurar a string real fora do repositorio.

### Opcao 1: user-secrets

No projeto da API:

```bash
cd ToyCompany.Api
dotnet user-secrets set "ConnectionStrings:OracleConnection" "User Id=SEU_USUARIO;Password=SUA_SENHA;Data Source=localhost:1521/XEPDB1;"
```

### Opcao 2: variavel de ambiente

PowerShell:

```powershell
$env:ConnectionStrings__OracleConnection="User Id=SEU_USUARIO;Password=SUA_SENHA;Data Source=localhost:1521/XEPDB1;"
```

O ASP.NET Core usa essa configuracao automaticamente e nao exige que a senha fique salva no Git.

### Script da tabela

```sql
CREATE TABLE TDS_TB_BRINQUEDOS (
    ID_BRINQUEDO NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    NOME_BRINQUEDO VARCHAR2(100) NOT NULL,
    TIPO_BRINQUEDO VARCHAR2(60) NOT NULL,
    CLASSIFICACAO NUMBER(2) NOT NULL,
    TAMANHO VARCHAR2(30) NOT NULL,
    PRECO NUMBER(10,2) NOT NULL
);
```

## Como executar

1. Abra a solucao `ToyCompany.sln` no Visual Studio.
2. Configure a `OracleConnection` via `user-secrets` ou variavel de ambiente.
3. Confirme se a tabela `TDS_TB_BRINQUEDOS` existe no Oracle.
4. Defina `ToyCompany.Api` como projeto de inicializacao.
5. Execute a API.

URLs configuradas:

- HTTP: `http://localhost:5184`
- HTTPS: `https://localhost:7184`
- Swagger: `https://localhost:7184/swagger` ou `http://localhost:5184/swagger`

## Endpoints

### GET `/brinquedos`

Retorna todos os brinquedos cadastrados.

### GET `/brinquedos/{id}`

Retorna um brinquedo especifico pelo identificador.

### POST `/brinquedos`

Exemplo de JSON:

```json
{
  "nomeBrinquedo": "Carrinho Turbo",
  "tipoBrinquedo": "Veiculo",
  "classificacao": 8,
  "tamanho": "Medio",
  "preco": 89.90
}
```

### PUT `/brinquedos/{id}`

Exemplo de JSON:

```json
{
  "nomeBrinquedo": "Carrinho Turbo X",
  "tipoBrinquedo": "Veiculo",
  "classificacao": 10,
  "tamanho": "Grande",
  "preco": 99.90
}
```

### DELETE `/brinquedos/{id}`

Remove o brinquedo informado.
